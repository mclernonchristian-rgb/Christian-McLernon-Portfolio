The Problem

Trend reports are yesterday’s news by the time they publish.

The standard workflow for identifying emerging trends — weekly social listening reports, agency briefs, platform trend pages — runs on a lag. By the time a trend makes it into a deck, it’s already past peak. For a 18–27 UK demographic that moves at TikTok speed, that lag is the difference between leading a conversation and chasing it.

I built the Trend Matrix to solve this for myself first. A tool that ingests live data across multiple platforms, scores trends by composite signal, and surfaces what’s about to break through — before it does.

A Python FastAPI backend handles every stage automatically. SQLite stores all trend data locally — no hosted database, no monthly bill, no moving parts. APScheduler fires the full pipeline at 6am UTC daily. The frontend is vanilla JS with Chart.js, served directly from the same FastAPI process.

Scoring Model

A composite score built for the 18–27 UK demographic.

Each trend is scored on a weighted composite across five dimensions. The weights aren’t arbitrary — they reflect how the 18–27 UK audience actually discovers and adopts trends: volume-led, but increasingly driven by social velocity and peer signal over editorial coverage.

# Composite trend score (0–100)
score = (
  volume * 0.40 # raw search interest — baseline awareness
  + velocity * 0.30 # 7-day growth rate — early-stage signal
  + reddit * 0.15 # community discussion — leading indicator
  + news * 0.10 # editorial coverage — niche to mainstream
  + demo_fit * 0.05 # UK 18–27 platform alignment
)

The backward pass — scorer.backtest_predictions() — resolves seven-day-old predictions against actual outcomes, building an accuracy log that improves the model over time.

Data Sources

Five inputs. One signal.

Each source contributes a different dimension of trend intelligence. Together they form a composite picture no single platform can offer — volume from Google, cultural velocity from Reddit, editorial weight from news, social-first signal from TikTok.

Google Trends
  • pytrends library
  • No API key required
  • UK region (geo=GB)
  • 7-day + 90-day series
Reddit
  • PRAW (Reddit API)
  • Subreddit velocity scoring
  • Comment + upvote signals
  • UK-focused community filter
NewsAPI + TikTok
  • NewsAPI — editorial weight
  • RSS fallback if key missing
  • TikTok stub via RapidAPI
  • Activates with key in .env

Why I Built It

A strategist who can build their own tools is a different kind of asset.

The Trend Matrix isn’t a portfolio piece dressed up as a project. It’s a tool I actually use. It started as a way to answer a question I kept running into while building campaign strategies: “How do I know this trend is still rising and hasn’t already peaked?”

Building it required learning FastAPI, SQLite schema design, async scheduling, and enough statistics to implement confidence-scored predictions. The result sits at the intersection of marketing strategy and data engineering — and makes both sharper.
